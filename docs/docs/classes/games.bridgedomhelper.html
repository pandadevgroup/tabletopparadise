Protected _ready

_ready: boolean = false

Inherited from DomHelper._ready

  • Defined in tabletop/dom-helper.ts:38

A flag for whether or not to render DOM changes (updateEl, class changes, etc).

Call DomHelper.ready() to set this to true.

When a user rejoins a game, all the previous game actions need to be played back to update the user's state. During this time, a large number of unnecessary DOM manipulations will occur. We prevent this by using the ready flag.
